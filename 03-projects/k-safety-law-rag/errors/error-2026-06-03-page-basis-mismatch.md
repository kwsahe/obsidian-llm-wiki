# 이슈: 외부 평가자가 페이지 번호를 "환각"으로 오판 (평가 기준 불일치)

## 발생 환경
- 프로젝트: K-Safety Law RAG
- 날짜: 2026-06-03
- 스택: Gemini 평가(`test3_gemini`), 원문 PDF vs 건설현장 한정 발췌본

## 에러 메시지
```
(런타임 에러 아님 — 외부 평가 오판)
Gemini가 p.82, p.94~96, p.115~116 등의 페이지 번호를 반복적으로
"페이지 환각(hallucination)"으로 지적
```

## 원인
시스템은 두 가지 서로 다른 페이지 체계를 동시에 다루고 있었다.
- table DB에 저장된 **원문 PDF 페이지** (예: 시행규칙 p.82)
- 건설현장 한정 **발췌본 PAGE 번호** (예: PAGE 7~8)

Gemini는 발췌본 PAGE 기준으로 답을 평가했지만, RAG 시스템은 원문 PDF 페이지 메타데이터를 사용해 답변했다. 실제로는 코드 버그가 아니라 **두 평가 주체가 서로 다른 페이지 체계를 기준으로 삼은 불일치**였다.

## 해결 방법
코드를 고치는 대신 원인을 분석해 "버그 아님"으로 판단했고, 후속 과제로 페이지 표기 방식을 다음 중 하나로 정리하기로 결정했다.
```
1) 원문 PDF 페이지 기준으로 통일, 또는
2) 원문 p.82 / 발췌본 p.7~8을 병기
```
