# 아이디어: 경쟁 상품 비교 & 구매 가이드 자동화 (AI 직원)

## 한 줄 요약
네이버쇼핑 API 기반으로 카테고리별 경쟁 상품을 수집해 기능별 우위를 비교·추천하고, 가격 변동을 주기적으로 추적해 "지금 사도 되는지"까지 조언하는 구매 가이드 자동화 프로그램.

https://semo.world/ — 역할별 "AI 직원"에게 업무를 위임하는 서비스. 이 중 "경쟁사 분석" 역할 컨셉을 가져오되, 크롤링 대신 공식 API 기반으로 구현 방향을 잡음.

## 왜 해볼만한가
- 공식 API(네이버 쇼핑 검색, 데이터랩)만으로 데이터 확보가 가능해 법적 리스크 없이 완결된 프로젝트를 만들 수 있다
- RAG+LLM(비교/추천 생성), 자동화(가격 추적 스케줄러), 데이터 엔지니어링(시계열 적재) 세 역량을 한 프로젝트로 보여줄 수 있다
- O2O 프로젝트의 "API 자동 수집 → 분석 → 대시보드" 패턴을 새 도메인에 재사용할 수 있어 개발 속도가 빠르다
- "지금 사도 되는지" 같은 실사용 가치가 명확해서 데모 설득력이 높다

## 필요 기술/스택 후보
- 네이버 검색 API(쇼핑) — 경쟁 상품명/가격/판매처/브랜드 수집
- 네이버 데이터랩 API(쇼핑인사이트) — 카테고리 트렌드, 연령대/성별 관심도
- 스케줄러(APScheduler/cron) — 가격을 주기적으로 수집해 시계열로 적재
- DB(SQLite/PostgreSQL) — 상품별 가격 이력 저장
- LLM — 기능별 비교, 구매 가이드 문구, 가격 추이 기반 "지금이 살 때인지" 판단
- Flask 대시보드 — 비교 테이블, 가격 추이 그래프 (O2O 프로젝트 UI 패턴 재사용 가능)

## 예상 난이도 / 소요 시간
중간. API 연동과 가격 추적 자체는 O2O 프로젝트 경험으로 빠르게 구현 가능하지만, "기능별 비교"에 필요한 상세 스펙 데이터가 병목이다 — 네이버쇼핑 검색 API는 상품명·가격·판매처만 주고 배터리/무게 같은 상세 스펙은 주지 않는다. 카탈로그 상세 페이지를 파싱해야 하거나, MVP 단계에서는 소수 제품의 스펙을 수동으로 입력해 데모하는 게 현실적이다.

## Claude 의견
- **데이터는 확보 가능**: 경쟁 상품 가격·판매처·트렌드는 네이버 쇼핑 검색 API + 데이터랩 API로 크롤링 없이 확보 가능하다. 이전에 우려했던 "크롤링 우회" 리스크는 이 스코프에서는 해소됐다.
- **병목은 "기능별 비교"**: 공식 API가 상세 스펙을 주지 않기 때문에 비교 품질은 스펙 데이터 확보 방법에 달려있다. MVP는 카테고리 1개 + 제품 5~10개 정도로 좁히고 스펙은 수동 큐레이션으로 시작해서 LLM 비교/추천 로직부터 검증하는 걸 추천한다. 스펙 자동 수집은 이후 확장 과제로 남겨둔다.
- **가격 추적이 진짜 차별화 포인트**: 단순 "지금 가격 비교"는 다나와/에누리가 이미 잘한다. 여기에 "지난 N일간 가격 추이 + LLM의 매수 타이밍 판단"을 더하면 기존 가격비교 서비스와 차별화되는 지점이 생긴다.
- **겹침 재검토**: 이전에 지적했던 "로컬 RAG 프로그램"과의 매출분석 겹침은 스코프가 "구매자 대상 비교/추천"으로 명확해지면서 대부분 해소됐다. 남은 접점은 "LLM 비교/추천 생성 로직" 정도라 재사용 관계로 보면 된다.
