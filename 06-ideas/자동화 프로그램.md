## Claude 의견
- **강점**: 크롤링 → 분석 → 추천으로 이어지는 엔드투엔드 자동화 스토리는 "AI가 실무를 대신한다"는 메시지가 강해서 포트폴리오 임팩트가 크다.
- **리스크 1 (이미 본인이 지적함)**: 실제 기업 데이터가 없어 분석 정확도를 검증하기 어렵다는 점, 동의한다.
- **리스크 2 (중요)**: "크롤링 우회기술"을 스킬로 강조하는 건 주의가 필요하다. 대상 사이트 이용약관 위반이나 정보통신망법 등 법적 이슈로 이어질 수 있어서, 포트폴리오·면접 자리에서 오히려 감점 요인이 될 수 있다. 크롤링 우회보다는 오픈마켓 셀러 API, 공공데이터포털, 네이버 검색광고 API 등 공식 API를 우선 사용하는 걸 추천한다.
- **겹침**: "매출 분석" 부분이 "로컬 RAG 프로그램" 아이디어와 겹친다. 두 아이디어를 합치거나(이 프로젝트는 마케팅 추천에 집중, RAG 프로그램은 문서 QA에 집중), 역할을 분리해서 스코프 충돌을 정리할 필요가 있다.
- **추천 MVP**: 특정 오픈마켓 카테고리의 공개 데이터(가격/리뷰/트렌드)를 공식 API로 수집 → 트렌드 분석 → LLM 마케팅 문구/타겟팅 추천. "크롤링 우회"보다 "공식 API 조합"을 기술 스택으로 내세우는 게 안전하고 실무에도 더 가깝다.
